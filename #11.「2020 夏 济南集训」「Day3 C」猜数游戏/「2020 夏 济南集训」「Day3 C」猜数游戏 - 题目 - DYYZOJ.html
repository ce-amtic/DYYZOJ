题目描述

这是一道交互题。 这应该是出题人在比赛中放的第一道交互， 虽然貌似并不难， 还是祝大家玩的愉快

问题描述

liuzhangfeiabc 会随机从 中选择一个非负整数， 你需要猜出这个数是多少。你每次可以向 liuzhangfeiabc 提问一个 的非空子集， liuzhangfeiabc 会回答这个数是否在这个子集中。

(选手内心 os:这不直接二分就完了吗这么无聊的题还好意思放在 t3)

(liuzhangfeiabc 露出了邪魅的笑容)由于 liuzhangfeiabc 比较毒瘤心机， 他当然不打算让你直接猜出来， 所以他并不会保证给你的回答是真话， 换句话说他可能会说谎！然而这样你显然就没得玩了对吧， 所以他有一个额外的限制条件——他保证在任意连续 次询问中至少会有一句真话.

当然即使这样你还是很难问出这个具体的答案， 因此你的胜利条件有所放宽——你只需要给出一个集合， 使得答案在这个集合当中就算你赢。 集合的大小由你自己决定， 当然这是有一定限制的。

如果你能赢 liuzhangfeiabc 的话， 你就会收到来自 liuzhangfeiabc 的祝福呢~

实现细节

你不需要，也不应该实现主函数，你只需要实现函数 vector<int> guess(int testid,int n,int k)

其中 是子任务编号， 和 的含义如题面描述。 这个函数的返回值是一个 vector， 大小应在 之间， 其中出现的元素应在 之间且互不相同， 具体顺序随意。

你可以调用函数 bool query(vector<int> chk)这个函数接受一个 vector， 表示你这一次询问的元素集合， 大小应在 1~n 之间，其中出现的元素应在 0~n-1 之间且互不相同， 具体顺序随意。

这个函数返回一个 bool， 表示被猜的数是否在询问的集合中， 当然可能按照题面描述中的规则进行说谎。

你的程序中， 应当包含此题的头文件：#include "guess.h"

数据范围与评分标准

本题共 10 个子任务， 每个子任务由多个测试点组成， 每个子任务满分均为 10分， 实际得分为其中所有测试点得分的最小值.

对于每一个测试点， 如果你的程序不能正常结束， 或者没有返回正确的集合， 或者返回集合的大小超过相应的限制，或者调用 query 函数的次数超过相应的限制，或者调用 query 函数的参数不合法， 将得 0 分。 否则， 你的得分将根据该子任务的相关参数来确定。
